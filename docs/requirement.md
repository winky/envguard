# 要件定義

**バージョン:** 1.3 **対象プラットフォーム:** macOS / Linux（Windows は将来拡張） **実装言語:** Go（外部依存なし） **想定利用者:** AIツールをローカル起動する前の開発者本人

---

## 1. 概要

ローカル環境に存在する「Claude Code のサブプロセスに継承され得るクラウド／業務系の認証情報」を網羅的に検出し、リスクレベル付きでレポートするコマンドラインツール。macOS および Linux で動作する。**読み取り専用**であり、検出した情報の自動削除や修正は一切行わない。

ツール名: `envguard`

---

## 2. 目的と背景

### 2.1 背景

Claude Code は起動時に親シェルの環境変数をそのまま継承する。そのため、`~/.zshrc` などで `export AWS_ACCESS_KEY_ID=...` のように業務用クレデンシャルを恒常的にロードしていると、Claude Code のセッションからそれらの変数が参照可能になる。また、環境変数に載っていなくても `~/.aws/credentials` などの SDK 既定パスにあるファイルは SDK が自動で読むため、同様のリスクがある。

### 2.2 目的

1. Claude Code 起動前に**何が露出し得るかを可視化**する
2. \*\*どこに（環境変数 / ファイル / OS 固有の永続化機構 / .envrc）\*\*存在するかを特定する
3. 各検出項目に**リスクレベル**を付けて、対策判断を支援する
4. **対策案**（env -i 起動、direnv、aws-vault 等への移行）を提示する

### 2.3 非目的

- 実際のクレデンシャル値の表示（値は常にマスクする）
- OS のシークレットストア（Keychain 等）内シークレットの復号・取得
- 自動での削除・変更（すべてユーザーの判断に委ねる）
- Windows サポート（将来拡張余地として設計はするが初版は対象外）

## 3. スコープ

### 3.1 検出対象

| \# | 対象 | 理由 |
| --- | --- | --- |
| 1 | 現在のシェルの環境変数 | Claude Code がそのまま継承する |
| 2 | シェル設定ファイル（`~/.zshrc`, `~/.zshenv`, `~/.zprofile`, `~/.bashrc`, `~/.bash_profile`, `~/.profile`, `/etc/zshenv`, `/etc/zprofile`, `/etc/profile`） | 新規シェルで自動 export される |
| 3 | クラウド SDK のデフォルト認証ファイル（`patterns.md` セクション 2 参照） | SDK/CLI が環境変数なしで読みに行く |
| 4 | `launchctl` の setenv 変数（macOS のみ） | GUI アプリ含め全プロセスに継承される |
| 5 | `~/Library/LaunchAgents/*.plist` 内の環境変数定義（macOS のみ） | 起動エージェント経由で恒常的にセットされる |
| 6 | ホーム配下の `.envrc`（direnv）ファイル | ディレクトリ移動時に自動 export される |

### 3.2 検出しないもの

- パスワードマネージャ内のシークレット
- ブラウザのクッキー・セッションストレージ
- 動作中プロセスのメモリ内容
- fish shell の設定ファイル（`~/.config/fish/config.fish`）※制限事項として後述

### 3.3 制限事項

| 制限 | 理由 |
| --- | --- |
| **fish shell 未対応** | 初版スコープ外。将来拡張として `scanners/shell_config.py` に追加可能な構造にする |
| `.envrc` **内の動的構文非検出** | `export TOKEN=$(cat ~/.token)` などのコマンド置換・heredoc はパース不可。正規表現による静的解析のみ実施し、制限をレポートに注記する |
| **launchctl / LaunchAgents は macOS 専用** | macOS 以外の OS では両スキャナを自動スキップする。Linux 向け代替（systemd environment 等）は将来拡張 |
| **launchctl の macOS バージョン依存** | `launchctl getenv` は macOS 12 Monterey 以降を前提とする |

## 4. 機能要件

### FR-1: 環境変数スキャン

- `os.environ` を走査し、キー名がクラウド系パターンにマッチする変数を列挙
- 値は常にマスキング（FR-9 参照）

### FR-2: シェル設定ファイルスキャン

- セクション 3.1 の 2 に列挙したファイルを読み込み、`export FOO=bar` / `FOO=bar` / `setenv FOO bar`（csh系）パターンを行番号付きで抽出
- キー名がクラウド系パターンにマッチするもののみ報告
- コメント行（`#` で始まる行）は除外
- `export` を伴わない `FOO=bar` 形式は `note` に「export なし（シェル関数内ローカル変数の可能性あり）」を付記する

### FR-3: 認証ファイル検出

- `patterns.md` セクション 2 の既定パスにファイル／ディレクトリが存在するかを確認
- 存在する場合、ファイルサイズと最終更新日を取得
- 中身は一切読まない
- ディレクトリの場合はディレクトリ自体の存在のみ確認

### FR-4: launchctl チェック（macOS 専用）

- macOS 以外の OS では自動スキップし、サマリに「skipped (not macOS)」と表示
- `patterns.md` セクション 3 の候補リストについて `launchctl getenv VAR` を順次実行
- 値が返ればマスクして報告
- `launchctl` コマンドが存在しない場合はスキャナ全体をスキップし、サマリに「skipped」と表示

### FR-5: LaunchAgent plist スキャン（macOS 専用）

- macOS 以外の OS では自動スキップし、サマリに「skipped (not macOS)」と表示
- `~/Library/LaunchAgents/*.plist` を走査
- `<key>EnvironmentVariables</key>` ブロック内のキー名を抽出
- キー名がクラウド系パターンにマッチするもののみ報告

### FR-6: direnv `.envrc` 検出

- `$HOME` 配下（深さ 4 階層まで）を走査し `.envrc` を列挙
- 除外ディレクトリ: `node_modules`, `.git`, `Library`, `Applications`, `Movies`, `Music`, `Pictures`, `Downloads`, `.Trash`
- 各 `.envrc` の内容を FR-2 と同じルールでスキャン
- 走査時間上限: 10 秒（タイムアウトしたら部分結果で継続、警告を表示）
- `direnv` コマンドは呼ばない。ファイル内容の正規表現解析のみ実施

### FR-7: リスク分類

| レベル | 判定基準 | 色（ANSI） |
| --- | --- | --- |
| **CRITICAL** | 長期有効な秘密鍵・シークレット（`*_SECRET_ACCESS_KEY`, `*_PRIVATE_KEY`, `*_SECRET`, DB パスワード等） | 赤 |
| **HIGH** | 本番アクセス可能なトークン類・一時認証情報（`*_TOKEN`, `*_API_KEY`, `AWS_SESSION_TOKEN` 等） | マゼンタ |
| **MEDIUM** | アクセスキー ID・設定情報（`AWS_ACCESS_KEY_ID`, `AWS_PROFILE`, `GOOGLE_APPLICATION_CREDENTIALS` のパス等） | 黄 |
| **LOW** | リージョン・エンドポイント等の非機密設定（`AWS_REGION`, `AWS_DEFAULT_OUTPUT` 等） | シアン |
| **INFO** | 認証ファイルの存在のみ（中身は見ていない） | 白 |

### FR-8: 出力フォーマット

- **テキスト形式（デフォルト）**: セクション分けされた人間可読レポート、色付き、サマリ付き
- **JSON 形式（**`--json`**）**: マシン可読、CI 連携や差分比較用
- **サマリのみ（**`--summary`**）**: 件数のみ

詳細は `cli-spec.md` を参照。

### FR-9: マスキング

値を表示する必要がある場合は以下の規則で必ずマスクする。平文値は内部の中間変数にも保持しない。

| 値の長さ | 表示 |
| --- | --- |
| 0〜8 文字 | `****`（全マスク） |
| 9 文字以上 | `****` + 末尾 4 文字 |

例: `AKIA1234567890ABCDEF`（20文字）→ `****CDEF` / `short123`（8文字）→ `****`

### FR-10: 対策提案

レポート末尾に、検出結果に応じた対策を自動で提示する。

- 環境変数がヒット → `env -i` 起動例、direnv 設定例
- 認証ファイルがヒット → `aws-vault` / 1Password CLI への移行案内
- LaunchAgent がヒット → 該当 plist のパスと確認手順

## 5. 非機能要件

| 項目 | 要件 |
| --- | --- |
| 依存 | Go 標準ライブラリのみ（外部パッケージ不要） |
| 配布 | クロスコンパイルによるシングルバイナリ（Go のインストール不要） |
| 対応 OS | macOS 12 Monterey 以降、主要 Linux ディストリビューション（Ubuntu 20.04+、Debian 11+ 等）。Windows は将来拡張 |
| 実行時間 | 通常 5 秒以内（`.envrc` 走査含めて最大 15 秒） |
| メモリ | 100MB 以下 |
| 副作用 | 一切の書き込み・削除・ネットワークアクセスなし |
| ログ | stderr に進捗表示（`--quiet` で抑止可能） |
| 終了コード | `0`: 何も検出されず / `1`: 検出あり / `2`: 実行エラー |
| 文字コード | UTF-8 |
